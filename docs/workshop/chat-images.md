## 1. 이미지 활용하기

Kiro는 채팅 세션 내에서 이미지를 직접 분석하고 논의할 수 있습니다.

이미지는 터미널에 드래그 앤 드롭하거나, read 도구(fs\_read) 이미지 모드(Image Mode) 를 통해 전달할 수 있습니다.

***

### 1.1. 이미지 드래그 앤 드롭

가장 간단한 이미지 공유 방식은 이미지를 터미널 창으로 직접 드래그 앤 드롭하는 것입니다.

드래그 앤 드롭 시:

* 이미지 파일 경로가 프롬프트에 자동 삽입됨
* 필요 시 추가 설명을 입력하여 Kiro에게 수행할 작업을 지정
* Kiro는 이미지를 처리한 뒤 해당 내용에 기반하여 응답 생성

#### Example

```
Kiro> /path/to/architecture-diagram.png 이 아키텍처(architecture)를 설명해주고, 이를 구현하기 위한 샘플 코드(sample code)를 생성해 줄 수 있나요?
```

***

### 1.2. `fs_read` 도구를 활용한 이미지 분석

이미지를 보다 명확하게 지정하고 싶다면 read 도구(fs\_read) 를 직접 사용할 수 있습니다:

```
Kiro> /path/to/screenshot.png 에 있는 스크린샷(screenshot)을 분석할 수 있나요?
```

Kiro는 대화 중 이미지 파일 경로나 확장자를 언급하면 Image Mode 사용을 자동 제안합니다.

***

## 2. 이미지 활용 사례

Kiro는 다양한 이미지 기반 작업을 지원하며, 대표적인 활용 시나리오로는 다음이 있습니다:

* 오류 메시지가 포함된 스크린샷 분석과  문제 원인 파악 및 해결 전략 제안
* 아키텍처 다이어그램 → 코드 구현 변환과 시스템 구조 해석 및 샘플 코드 생성
* UI/UX 디자인 논의 및 HTML/CSS 생성 및 와이어프레임·모형(Mockup)을 코드로 변환
* 플로우차트(Flowchart) 이해 및 알고리즘 변환
* 이미지 형태로 공유된 코드 스니펫 리뷰
* 기술 다이어그램 분석 후 문서화 지원

***

## 3. 지원 포맷 및 제한 사항

#### 지원 이미지 형식

* JPEG/JPG
* PNG
* GIF
* WebP

#### 제한 사항

* 이미지 크기: 10MB 이하
* 한 번의 요청에서 최대 10개 이미지 공유 가능

***

## 4. 모범 사례

최상의 결과를 위해 아래 사항을 권장합니다:

* 텍스트가 포함된 경우 고해상도 이미지 사용
* 이미지 분석 목적을 명확히 지시
* 복잡한 다이어그램의 경우 추가 설명을 제공하여 정확도 향상
